# 구구단 산성비 게임 PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 제품명
**구구단 산성비 (Multiplication Rain)**

### 1.2 목적
초등학교 2학년 학생들이 곱셈구구를 재미있게 학습할 수 있도록 게임화된 학습 도구를 제공한다.

### 1.3 타겟 사용자
- 초등학교 2학년 학생 (7-8세)
- 곱셈구구를 처음 배우거나 연습이 필요한 학생
- 부모 및 교사 (학습 진행 상황 모니터링)

### 1.4 핵심 가치
- 학습과 게임을 결합한 즐거운 교육 경험
- 반복 학습을 통한 구구단 암기 효과
- 즉각적인 피드백으로 학습 동기 부여

---

## 2. 게임 컨셉

### 2.1 기본 개념
하늘에서 곱셈 문제(예: 3×4)가 적힌 물방울이 떨어진다. 플레이어는 화면 하단에서 정답(12)을 입력하거나 선택하여 물방울을 터뜨려야 한다. 물방울이 바닥에 닿으면 생명이 줄어든다.

### 2.2 게임 플레이 방식
1. 화면 상단에서 곱셈 문제가 적힌 물방울이 천천히 떨어진다
2. 플레이어는 키보드로 정답을 입력하거나 화면의 숫자 버튼을 클릭
3. 정답을 맞추면 해당 물방울이 터지고 점수 획득
4. 틀리면 입력이 취소되고 다시 입력 가능
5. 물방울이 바닥에 닿으면 생명 1개 감소
6. 생명이 모두 소진되면 게임 종료

## 3. 주요 기능 명세

### 3.1 게임 모드

#### 3.1.1 연습 모드 (Practice Mode)
- **단계 선택**: 2단부터 9단까지 원하는 단 선택
- **난이도**: 쉬움 (물방울 속도 느림, 한 번에 1-2개)
- **제한 시간**: 없음
- **목적**: 특정 단을 집중 연습

#### 3.1.2 스테이지 모드 (Stage Mode)
- **진행 방식**: 2단부터 시작하여 순차적으로 단계 상승
- **클리어 조건**: 각 단마다 10문제 이상 맞추기
- **난이도 상승**: 단계가 올라갈수록 물방울 속도 증가
- **생명**: 스테이지당 5개

#### 3.1.3 챌린지 모드 (Challenge Mode)
- **문제 범위**: 2단~9단 랜덤 출제
- **난이도**: 어려움 (물방울 속도 빠름, 여러 개 동시 출현)
- **목표**: 최고 점수 달성
- **생명**: 3개

### 3.2 게임 UI 요소

#### 3.2.1 메인 화면
- 게임 타이틀 및 로고
- 모드 선택 버튼 (연습/스테이지/챌린지)
- 설정 버튼
- 점수 기록 (최고 기록)
- 학습 통계 보기

#### 3.2.2 게임 화면
- **상단 정보 바**
  - 현재 점수
  - 남은 생명 (하트 아이콘)
  - 현재 단 (연습 모드)
  - 일시정지 버튼
  
- **게임 영역**
  - 물방울이 떨어지는 메인 화면
  - 물방울 내부에 큰 글씨로 곱셈 문제 표시 (예: "3 × 4")
  
- **입력 영역**
  - 숫자 입력 칸 (크게 표시)
  - 숫자 버튼 (0-9, 지우기, 제출)
  - 또는 키보드 입력 지원

#### 3.2.3 결과 화면
- 총 점수
- 정답 개수 / 전체 문제 수
- 정확도 (%)
- 획득한 별/배지
- 버튼: 다시하기, 메인으로

### 3.3 학습 지원 기능

#### 3.3.1 힌트 시스템
- 물방울이 중간쯤 내려오면 정답의 일의 자리 힌트 표시 (선택적)
- 힌트 사용 시 점수 감소

#### 3.3.2 피드백 시스템
- 정답: 긍정적인 효과음 + 파티클 효과 + "잘했어요!" 메시지
- 오답: 부드러운 효과음 + "다시 생각해봐요" 메시지
- 연속 정답: 콤보 표시 및 보너스 점수

#### 3.3.3 학습 통계
- 단별 정답률 그래프
- 가장 어려워하는 문제 분석
- 총 플레이 시간 및 문제 해결 수
- 주간/월간 진행 상황

### 3.4 게임 설정

#### 3.4.1 난이도 조절
- **쉬움**: 물방울 속도 느림, 동시 출현 1-2개, 생명 5개
- **보통**: 물방울 속도 중간, 동시 출현 2-3개, 생명 3개
- **어려움**: 물방울 속도 빠름, 동시 출현 3-5개, 생명 3개

#### 3.4.2 사운드 설정
- 배경음악 ON/OFF
- 효과음 ON/OFF
- 볼륨 조절

#### 3.4.3 게임 옵션
- 입력 방식: 키보드 / 화면 버튼 / 둘 다
- 힌트 표시 ON/OFF
- 색상 테마 선택 (밝은/어두운)

## 4. 비기능적 요구사항

### 4.1 디자인 원칙
- **어린이 친화적**: 밝고 귀여운 색상, 큰 글씨, 직관적인 아이콘
- **가독성**: 구구단 문제는 최소 24pt 이상의 굵은 글씨
- **접근성**: 색맹을 고려한 색상 조합, 명확한 피드백

### 4.2 성능 요구사항
- 게임 로딩 시간: 3초 이내
- 입력 반응 속도: 즉각적 (100ms 이내)
- 프레임률: 최소 30fps 유지
- 모바일/태블릿/PC 모두 지원

### 4.3 보안 및 개인정보
- 학습 데이터는 로컬 저장 (선택적 클라우드 백업)
- 개인정보 수집 최소화
- 부모 동의 절차 (클라우드 저장 시)

### 4.4 호환성
- **플랫폼**: 웹 브라우저 (Chrome, Safari, Edge)
- **디바이스**: PC, 태블릿, 스마트폰
- **화면 크기**: 반응형 디자인 지원

---

## 5. 게임 메커니즘 상세

### 5.1 점수 시스템
- 기본 점수: 정답 1개당 10점
- 콤보 보너스: 연속 정답 시 추가 점수
  - 5연속: +20점
  - 10연속: +50점
  - 15연속: +100점
- 속도 보너스: 빠르게 답할수록 추가 점수 (1-5점)
- 완벽한 스테이지 클리어: +200점

### 5.2 물방울 속도 조절
- **레벨 1-3**: 5초 동안 화면을 횡단
- **레벨 4-6**: 4초 동안 화면을 횡단
- **레벨 7-9**: 3초 동안 화면을 횡단
- **챌린지 모드**: 2.5초

### 5.3 보상 시스템
- **별 획득**: 스테이지 점수에 따라 1-3개 별
- **배지 시스템**:
  - "구구단 초보": 첫 스테이지 클리어
  - "2단 마스터": 2단 연습모드 30문제 연속 정답
  - "구구단 천재": 모든 단 마스터
  - "완벽주의자": 한 스테이지 무실수 클리어
  - "집중력 왕": 50콤보 달성

## 6. 기술 스택 (권장)

### 6.1 프론트엔드
- **프레임워크**: React.js 또는 Vue.js
- **게임 엔진**: Phaser.js 또는 PixiJS
- **스타일링**: CSS3 애니메이션, Tailwind CSS

### 6.2 백엔드 (선택적)
- **서버**: Node.js + Express (점수 저장용)
- **데이터베이스**: Firebase 또는 MongoDB (학습 통계)

### 6.3 기타
- **애니메이션**: GreenSock (GSAP)
- **사운드**: Howler.js
- **배포**: Vercel, Netlify, GitHub Pages

## 9. 위험 요소 및 대응

### 9.1 위험 요소
- 게임이 너무 어려워 학생들이 좌절할 수 있음
- 게임에만 집중하고 학습 효과가 떨어질 수 있음
- 모바일 환경에서 입력이 불편할 수 있음

### 9.2 대응 방안
- 세밀한 난이도 조절 및 충분한 테스트
- 학습 통계를 통해 효과 측정 및 부모 리포트 제공
- 터치 친화적 UI 설계 및 입력 방식 다양화
- 초등학교 2학년 학생 대상 사용성 테스트 필수
